# RE-14.2 synthesis — record-prefix filter isolates real records, ArcWall has 100% clean signal on Einhoven Partitions/5

**Date:** 2026-04-21
**Scope:** Apply the record-prefix filter (bytes at +2/+3 after tag == 0x0000) across all 80 tagged classes on three partition streams. Distinguish real records from UTF-16 text artifacts.

## TL;DR

The filter from D13/D14 works. Results split cleanly into three populations:

1. **Real records (kept ratio >50%):** `AnalyticalLevelAssociationCell`,
   `A3PartyAImage`, `ADTGridImportVocabulary`, `APIVSTAMacroElem*`,
   `AProperties`, `ADocWarnings`, `AnalyticalPanelPatternHelper`, `ArcWall`,
   `LinearConstraint`, and others. These tags correspond to genuine
   structured records.
2. **Bimodal (10-40% kept):** `HostObjAttr` (11-34%), `AbsDbViewPressureLossReport`
   (10-28%), `AppearanceAsset` (23%). Real records mixed with UTF-16 text
   coincidence.
3. **Pure text artifact (<5% kept):** `AbsCurveGStep` (0.9%). The letter 'a'
   in UTF-16 strings dominated its raw hit count.

**Most actionable finding:** `ArcWall` (tag 0x0191) on Einhoven Partitions/5
has **32 raw occurrences, 32 filtered — 100% kept ratio**. These are genuine
ArcWall architectural element records, which is a direct path to finishing
DEC-05 (IfcWall count > 0 on real file).

## Record structure: ArcWall as the cleanest path forward

With 32 ArcWall records at 100% clean signal, these are the prime
candidates for byte-level record decode:

- Direct tag 0x0191 (well-known from RE-09 F11).
- 100% kept — zero text artifacts, we know every one is real.
- Small enough sample (32) to manually inspect each one.
- High signal-to-noise ratio for reverse-engineering the record body.

**Immediate follow-up (RE-14.3):** hex-dump the 128 B following each of the
32 ArcWall occurrences on Einhoven Partitions/5. Compute distance-to-next-
ArcWall. With 32 records in 587 KB, average spacing is ~18 KB — so records
are likely variable-length with substantial content each.

Once the ArcWall record layout is understood, wire up:
- `elements::decoders::arc_wall::decode(&[u8]) -> ArcWall` (DEC-01/02)
- Route to `IFCWALL` in the IFC exporter (DEC-04)
- Integration test: `iter_partition_elements()` on Einhoven yields ≥32 walls (DEC-05)

## Hypotheses updated

**H10 ↑ (conf 0.9 → 0.95):** record-prefix filter (col0=0x00, col1=0x00) cleanly
separates real records from UTF-16 text artifacts. Validated on 3 streams.

**H11 ↑ (conf 0.5 → 0.8):** all top-signal tags are bimodal. Validated — every
10%+-kept-ratio tag fits.

**H12 (new, conf 0.6):** partition stream numbers correlate with element
categories. Partitions/0 = analytical/infrastructure, Partitions/5 = walls,
Partitions/46 = everything (2024 merged?). Specific mapping requires more
cross-partition analysis.

**H13 (new, conf 0.7):** variable-length record bodies. 32 ArcWalls in 587 KB
= ~18 KB average per record. This is large — suggests a Wall record includes
geometry or references to multiple geometry items.
